**Release date:** 2026-07-04

**Sprint:** Sprint 0 / Sprint 0.5 (Release Preparation)  
**Git tag:** `v0.1.0-alpha`  
**Branch:** `feature/platform-foundation`

---

## Summary

GovOS **v0.1.0-alpha** is the first official release of the Enterprise Government Platform backend. It delivers eight bounded-context modules as a modular monolith — a reusable platform foundation with no REST API, UI, or business-process modules yet.

This is an **alpha** release: suitable for architecture review, local development, and integration planning. It is **not** production-ready.

## Known Limitations

### Not included in this release

- REST controllers and OpenAPI-exposed business APIs
- Spring Security, JWT, OAuth2, or role-based HTTP enforcement
- Workflow execution engine, SLA scheduler, or BPMN integration
- Notification delivery (email, SMS, push) — provider stubs only
- Search indexing and full-text query (SRH module)
- Automatic audit capture via AOP or Hibernate listeners
- Angular workspace or any web UI
- Complaint (CMP) or other government-process modules
- Production deployment manifests (Kubernetes, etc.)
- Seed data or demo tenants

### Test coverage gaps

Service-layer unit tests exist for **MDM**, **IDM**, and **AUD** only. ORG, DOC, NTF, and WRK were built under Sprint 0 before merge quality gates and remain grandfathered.

### Branch state

This release is tagged from `feature/platform-foundation`, which has **not** yet been merged to `develop` or `main`.

## Verification

```powershell
cd backend
.\mvnw.cmd -pl govos-api -am clean verify
```

Expected: BUILD SUCCESS, 142 tests pass, JaCoCo coverage checks pass.

Local runtime (requires Docker PostgreSQL):

```powershell
.\mvnw.cmd -pl govos-api -am install -DskipTests
.\mvnw.cmd -pl govos-api spring-boot:run
```

Expected: Flyway at v1.7.0, 45 JPA repositories, app on port 8080.
